# Math SFT Datasets for Improving GSM8K — Research Notes

## Context

**Goal**: Find additional datasets for SFT on nanochat (830M params, d=26) to improve GSM8K performance.

**Current SFT mixture** (in `chat_sft.py`):
- SmolTalk (460K general conversations)
- Identity conversations (1K × 2 epochs)
- MMLU auxiliary train (100K × 3 epochs)
- GSM8K train (8K × 4 epochs = 32K effective)
- SimpleSpelling (200K)
- SpellingBee (80K)

## Detailed Entries

### 1. MetaMathQA
- **HuggingFace**: `meta-math/MetaMathQA` (also `meta-math/MetaMathQA-40K` for quick experiments)
- **License**: MIT
- **Size**: 395K rows
- **Description**: Augmented math Q&A pairs bootstrapped from GSM8K and MATH train sets using answer augmentation, question rephrasing, self-verification, and FOBAR (backward reasoning) via GPT-3.5.
- **Why it helps GSM8K**: Directly derived from GSM8K train questions with diverse rephrasings and augmented solution paths. MetaMath-7B achieves 66.5% on GSM8K (vs 14.6% base LLaMA-2-7B).
- **Citation**: Yu et al., "MetaMath: Bootstrap Your Own Mathematical Questions for Large Language Models", arXiv:2309.12284, 2023.
- **Format**: `{"query": ..., "response": ..., "type": ..., "original_question": ...}`. Plain text CoT responses. Easy conversion to nanochat JSONL.

### 2. Orca-Math 200K
- **HuggingFace**: `microsoft/orca-math-word-problems-200k`
- **License**: MIT
- **Size**: 200K rows
- **Description**: High-quality synthetic grade-school math word problems created using a multi-agent setup with GPT-4 Turbo. Problems generated by an "agent" and verified/refined by a "teacher" agent.
- **Why it helps GSM8K**: Specifically designed for small language models on grade-school math. A 7B model trained only on this 200K dataset surpassed LLaMA-2-70B and ChatGPT-3.5 on GSM8K.
- **Citation**: Mitra et al., "Orca-Math: Unlocking the potential of SLMs in Grade School Math", arXiv:2402.14830, 2024.
- **Format**: `{"question": ..., "answer": ...}`. Natural-language CoT, no tool calls. Clean conversion.

### 3. OpenMathInstruct-1
- **HuggingFace**: `nvidia/OpenMathInstruct-1`
- **License**: NVIDIA (permits commercial use)
- **Size**: 1.8M problem-solution pairs
- **Description**: Math instruction tuning dataset generated using Mixtral-8x7B. Solutions use a mix of text reasoning and Python code blocks, covering GSM8K and MATH training problems.
- **Why it helps GSM8K**: Multiple solution attempts per problem provide diverse reasoning paths. OpenMath-CodeLlama-70B trained on a subset achieves 84.6% on GSM8K. Built with a permissively-licensed generator (Mixtral).
- **Citation**: Toshniwal et al., "OpenMathInstruct-1: A 1.8 Million Math Instruction Tuning Dataset", arXiv:2402.10176, 2024 (NeurIPS 2024).
- **Format**: `{"question": ..., "generated_solution": ..., "expected_answer": ...}`. Some solutions contain Python code blocks. Large — would need subsampling.

### 4. DART-Math-Hard
- **HuggingFace**: `hkust-nlp/dart-math-hard`
- **License**: MIT
- **Size**: 585K rows
- **Description**: Created via Difficulty-Aware Rejection Tuning (DARS), which allocates more generation attempts to harder problems from GSM8K and MATH train sets.
- **Why it helps GSM8K**: Specifically oversamples problems that models struggle with, addressing the failure mode where easy problems dominate training. Competitive or superior to prior SOTA on GSM8K with smaller dataset sizes.
- **Citation**: Tong et al., "DART-Math: Difficulty-Aware Rejection Tuning for Mathematical Problem-Solving", arXiv:2407.13690, 2024 (NeurIPS 2024).
- **Format**: `{"query": ..., "response": ...}`. Clean text CoT. Also available: `dart-math-uniform` (591K, uniform sampling).

### 5. MathInstruct
- **HuggingFace**: `TIGER-Lab/MathInstruct`
- **License**: MIT
- **Size**: 262K rows
- **Description**: Curated compilation from 13 math rationale datasets (6 newly created) covering diverse mathematical fields. Uniquely combines both Chain-of-Thought (CoT) and Program-of-Thought (PoT) rationales. Sources include AQuA, GSM8K, MATH, NumGLUE, and newly synthesized datasets.
- **Why it helps GSM8K**: Hybrid CoT+PoT format teaches models to reason both verbally and programmatically. Breadth across 13 source datasets provides diverse problem formats. MAmmoTH-7B trained on this achieves strong GSM8K results.
- **Citation**: Yue et al., "MAmmoTH: Building Math Generalist Models through Hybrid Instruction Tuning", arXiv:2309.05653, 2023 (ICLR 2024).
- **Format**: `{"source": ..., "instruction": ..., "output": ...}`. Outputs are either CoT (text) or PoT (Python code), indicated by the `source` field. Could filter to CoT-only for simpler integration.

### 6. NuminaMath-CoT
- **HuggingFace**: `AI-MO/NuminaMath-CoT`
- **License**: Apache-2.0
- **Size**: ~860K rows
- **Description**: Large-scale math dataset with CoT solutions from Chinese high school exercises, university entrance exams, and international math olympiad problems. OCR'd from PDFs, translated to English.
- **Why it helps GSM8K**: Large volume of step-by-step word problems at high-school level overlaps with GSM8K's arithmetic/algebra domain. Foundation for AIMO competition-winning models.
- **Citation**: Li et al., "NuminaMath", Numina/Hugging Face, 2024.
- **Format**: `{"source": ..., "problem": ..., "solution": ...}`. Pure text CoT. Some problems harder than GSM8K level — consider filtering by source. May contain translation artifacts.

### 7. TinyGSM
- **HuggingFace**: `TinyGSM/TinyGSM`
- **License**: MIT
- **Size**: 12.3M rows
- **Description**: Massive synthetic dataset of grade school math problems paired with Python solutions, generated by GPT-3.5. Specifically designed to enable small models to achieve high GSM8K accuracy.
- **Why it helps GSM8K**: A 1.3B model fine-tuned on TinyGSM achieves 81.5% GSM8K accuracy — this is very close to nanochat's 830M scale. Explicitly created for the "small model + GSM8K" use case.
- **Citation**: Liu et al., "TinyGSM: achieving >80% on GSM8k with small language models", arXiv:2312.09241, 2023.
- **Format**: `{"question": ..., "code": ..., "answer": ...}`. Solutions are Python code, not text CoT. Very large — heavy subsampling (200K-500K) essential. Could convert Python to tool-call format.

### 8. OpenMathInstruct-2
- **HuggingFace**: `nvidia/OpenMathInstruct-2` (use `train_1M` config)
- **License**: NVIDIA (permits commercial use)
- **Size**: 14M total; pre-made subsets at 1M, 2M, 5M
- **Description**: Successor to v1, generated with Llama-3.1-405B-Instruct. Nearly 8× larger than previous largest open-source math reasoning dataset. Contains problems from GSM8K and MATH with many solution variants.
- **Why it helps GSM8K**: Higher quality solutions than v1 (405B vs Mixtral generator). Rich reasoning diversity with many solutions per problem. NVIDIA provides a contamination explorer for evaluation integrity.
- **Citation**: Toshniwal et al., "OpenMathInstruct-2: Accelerating AI for Math with Massive Open-Source Instruction Data", arXiv:2410.01560, 2024.
- **Format**: Similar to v1. Use `load_dataset("nvidia/OpenMathInstruct-2", "train_1M")`. Some solutions contain code blocks. Consider filtering to 200K-500K GSM8K-derived problems.

### 9. PersonaMathQA
- **HuggingFace**: `jingluo/PersonaMathQA`
- **License**: Not explicitly specified (research use)
- **Size**: ~129K rows
- **Description**: Created via persona-driven data augmentation on GSM8K and MATH problems. Each problem is rewritten from the perspective of diverse personas (students, teachers, professionals), with "reflection" steps on common mistakes.
- **Why it helps GSM8K**: Achieves SOTA GSM8K results despite being smaller than MetaMathQA or DART-Math, demonstrating exceptional data efficiency. Persona diversity and error-reflection mechanism help avoid common mistakes.
- **Citation**: Luo et al., "PersonaMath: Boosting Mathematical Reasoning via Persona-Driven Data Augmentation", arXiv:2410.01504, 2024.
- **Format**: Question-answer pairs. Compact size (~129K) makes it ideal as a direct addition without dominating other tasks. License should be verified before use.

### 10. WebInstructSub (math subset)
- **HuggingFace**: `TIGER-Lab/WebInstructSub`
- **License**: Apache-2.0
- **Size**: 2.39M total (filter for math subset)
- **Description**: Instruction-response pairs mined from Common Crawl using a recall-extract-refine pipeline. Covers math, science, and STEM domains. Real Q&A pairs from forums and educational sites, refined for quality.
- **Why it helps GSM8K**: Naturalistic distribution of math problems as they appear "in the wild," complementing formulaic synthetic datasets. MAmmoTH2-7B trained on this improves from 36% to 68.4% on GSM8K without domain-specific data.
- **Citation**: Yue et al., "MAmmoTH2: Scaling Instructions from the Web", arXiv:2405.03548, 2024 (NeurIPS 2024).
- **Format**: `{"question": ..., "answer": ...}`. Filter by metadata for math-related Q&A. Web-mined quality varies — some filtering/dedup recommended. Subsample 100K-300K math rows.

## Practical Notes for nanochat (830M params)

**Key constraints:**
- At 830M params, adding >500K math rows risks catastrophic forgetting of other skills
- Quality > quantity at this scale
- All datasets need conversion to nanochat's conversation format: `[{"role": "user", "content": "..."}, {"role": "assistant", "content": "..."}]`
- Datasets with Python code solutions can either be converted to nanochat's `{"type": "python", "text": ...}` tool-call format or filtered to text-only CoT

**Integration path:**
- Simplest: Convert dataset to JSONL, use `CustomJSON(filepath=...)` in `chat_sft.py`
- More robust: Write a new `Task` subclass in `tasks/` that loads from HuggingFace directly (like `SmolTalk` or `GSM8K`)
